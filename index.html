Regression mit Feed Forward Neural Network und TensorFlow.js

Ziel dieser Web Anwendung ist die Approximation einer unbekannten reellwertigen Funktion auf dem Intervall [-2, 2] mit einem Feed Forward Neural Network.

Diskussion

Das Clean Modell wurde auf unverrauschten Daten trainiert und approximiert die Ground Truth Funktion sehr gut. Der Testfehler kann trotzdem höher als der Trainingsfehler sein, weil das Modell nur 50 zufällige Trainingspunkte sieht und auf 50 unabhängigen Testpunkten bewertet wird.

Das Best Fit Modell wurde auf verrauschten Daten mit moderater Trainingsdauer trainiert. Ziel dieses Modells ist nicht, jedes einzelne verrauschte Trainingslabel exakt zu treffen, sondern die zugrunde liegende Struktur der Funktion möglichst gut zu lernen.

Das Overfit Modell wurde auf denselben verrauschten Trainingsdaten deutlich länger trainiert. Dadurch erreicht es typischerweise einen niedrigeren Trainings MSE als das Best Fit Modell, generalisiert aber schlechter auf unabhängige Testdaten.

Dieses Verhalten zeigt Overfitting: Das Modell passt sich stärker an die verrauschten Trainingslabels an, verbessert dadurch den Trainingsfehler, verliert aber Generalisierungsfähigkeit. Die Testdaten wurden nicht zum Training verwendet, sondern nur zur abschließenden Bewertung.

In einer realen Modellwahl würde zusätzlich ein Validierungsdatensatz verwendet werden, um Parameter wie die Anzahl der Epochen auszuwählen, ohne die unabhängigen Testdaten indirekt für die Optimierung zu benutzen.

Qualitätssicherung und Anforderungsprüfung

  • Es werden genau 100 zufällige x Werte gleichverteilt im Intervall [-2, 2] erzeugt.
  • Die Daten werden zufällig in 50 Trainingspaare und 50 Testpaare aufgeteilt.
  • Das künstliche Rauschen wird nur auf die y Labels addiert, nicht auf die x Werte.
  • Die Rauschvarianz beträgt 0.05, also ist die Standardabweichung sqrt(0.05).
  • Die Testdaten werden nicht zum Training verwendet, sondern nur zur Evaluation.
  • Alle dargestellten MSE Werte werden nach dem Training separat auf den passenden Datensätzen berechnet.
  • Das Clean Modell, das Best Fit Modell und das Overfit Modell sind drei getrennt trainierte Modelle.
  • Datensatz und Modelle können gespeichert und wieder geladen werden.
  • Die Seite erzeugt die Daten und Ergebnisse beim initialen Laden automatisch dynamisch im Browser.

Dokumentation

1) Technisch

Die Web Anwendung ist als browserbasierte Lösung umgesetzt und besteht aus drei getrennten Dateien: index.html für die Seitenstruktur und Dokumentation, style.css für Layout und Gestaltung sowie app.js für Datenerzeugung, Modelltraining, Evaluation und Visualisierung. Dadurch bleiben Darstellung, Inhalt und Programmlogik nachvollziehbar voneinander getrennt.

Für das maschinelle Lernen wird TensorFlow.js verwendet. Damit werden die Feed Forward Neural Networks im Browser erstellt, kompiliert, trainiert und für Vorhersagen genutzt. Für die initiale Darstellung der Abgabe werden vortrainierte TensorFlow.js Modelle aus dem Projektordner models/ geladen und direkt auf den erzeugten Datensätzen ausgewertet. Zusätzlich können im Entwicklungsmodus neu trainierte Modelle über das TensorFlow.js Speicherziel localstorage:// lokal im Browser gespeichert und erneut geladen werden.

Für die Visualisierung wird Plotly.js eingesetzt. Die Bibliothek erzeugt die Scatterplots der Trainings und Testdaten, die Linienplots der Modellvorhersagen sowie die Loss Verläufe während des Trainings. Dadurch werden Datensätze, Modellverhalten und Overfitting direkt auf der HTML Seite sichtbar.

Datensätze können im Local Storage des Browsers gespeichert und wieder geladen werden. Der feste Seed 42 sorgt dafür, dass die zufällige Datenerzeugung reproduzierbar ist und zu den vortrainierten Modellen passende Daten erzeugt werden. Die Anwendung benötigt keine zusätzliche Build Umgebung und kann über einen lokalen oder öffentlichen Webserver direkt im Browser ausgeführt werden.

2) Fachlich

Die Implementierung erzeugt mit einem seedbaren Zufallsgenerator genau 100 x Werte gleichverteilt im Intervall [-2, 2]. Für jeden x Wert wird mit der vorgegebenen Polynomfunktion der zugehörige y Wert berechnet. Dieser unverrauschte Datensatz bildet die Grundlage für die weiteren Experimente.

Anschließend werden die 100 Datenpaare zufällig in 50 Trainingsdaten und 50 Testdaten aufgeteilt. Aus dem unverrauschten Datensatz wird zusätzlich ein verrauschter Datensatz erzeugt. Dabei wird ausschließlich auf die y Labels normalverteiltes Rauschen mit Mittelwert 0 und Varianz 0.05 addiert. Die x Werte bleiben unverändert, sodass gezielt Label Rauschen simuliert wird.

Das verwendete Modell ist ein Feed Forward Neural Network mit zwei Hidden Layern und jeweils 100 Neuronen. In den Hidden Layern wird ReLU als Aktivierungsfunktion verwendet, während der Output Layer aus einem einzelnen linearen Neuron besteht. Als Fehlerfunktion wird Mean Squared Error genutzt. Trainiert wird mit dem Adam Optimizer, einer Lernrate von 0.01 und einer Batch Size von 32.

Verglichen werden drei getrennt trainierte Modelle. Das Clean Modell wird auf unverrauschten Trainingsdaten trainiert. Das Best Fit Modell wird auf verrauschten Trainingsdaten mit moderater Trainingsdauer trainiert, um möglichst gut zu generalisieren. Das Overfit Modell wird auf denselben verrauschten Trainingsdaten deutlich länger trainiert, damit es stärker auf das Rauschen der Trainingslabels passt.

Für die initiale Darstellung werden die vortrainierten Modelle geladen und separat auf den passenden Trainings und Testdaten ausgewertet. Die MSE Werte werden nicht aus gespeicherten Trainingslogs übernommen, sondern nach dem Laden der Modelle direkt auf den dargestellten Datensätzen berechnet. Dadurch passen die angezeigten Loss Werte direkt zu den jeweiligen Diagrammen.
